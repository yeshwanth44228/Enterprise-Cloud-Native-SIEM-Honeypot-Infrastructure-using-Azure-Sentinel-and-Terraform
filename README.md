# Enterprise Cloud-Native SIEM & Honeypot Infrastructure (Azure + Microsoft Sentinel)

## Overview

This project demonstrates the deployment of a Windows Server 2022 honeypot in Microsoft Azure and the implementation of a cloud-native SIEM solution using Microsoft Sentinel.

The objective was to collect, analyze, and visualize security telemetry generated by failed authentication attempts against an internet-facing Windows server.

---

## Technologies Used

* Microsoft Azure
* Windows Server 2022
* Microsoft Sentinel
* Log Analytics Workspace
* Azure Monitor Agent
* Kusto Query Language (KQL)

---

## Architecture

Internet → Windows Honeypot VM → Azure Monitor Agent → Log Analytics Workspace → Microsoft Sentinel → KQL Detection Queries → Dashboard

---

## Detection Queries

### Failed Login Detection

Detects failed authentication attempts (Event ID 4625).

### Top Failed Login Sources

Identifies systems generating the highest number of failed login events.

### Attack Timeline

Visualizes failed login activity over time.

---

## Key Outcomes

* Deployed Windows Server 2022 honeypot in Azure
* Configured Azure Monitor Agent for log collection
* Integrated Log Analytics Workspace with Microsoft Sentinel
* Created KQL threat hunting queries
* Detected Event ID 4625 failed authentication attempts
* Built monitoring dashboard using Azure Workbooks
